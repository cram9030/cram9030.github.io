## Paper Summary

The basic premise is that given a bounded input $\epsilon$, there will be a set of input features that will have the output bounded by $\delta$ relative to the unperturbed input. This set is defined as *irrelevant features* which is to say that perturbing that set does not change the outcoming results which the paper calls $x^B$. Then there will be a set that by adding that single additional member of the remaining set would cause the output to be larger than the bounding $\delta$ which is refered to as an *optimal* and named $x^A$. The definition of optimal here is interesting because it refers to the minimal additional feature added to the set $x^B$ that causes a change in the output to exceed the threshold $\delta$ when compared to the nonperturbed solution.

The authors propose to iteratively populate the irrelevant features by sampling a single feature at a time, then sorting it to $x^A$ or $x^B$. The animation below shows an example of the iterative process. While the animation makes it seem like the propagation of the perturbations is a single forward instance, it is in fact searching the bounded perturbed input. In the animation, the hashed features are those that are allowed to be perturbed, while the unhashed features are kept constant. The red hashed boxes represent features that have been added to the optimal set, which is then kept constant.

While the majority of the paper demonstrates how the proposed solution works on various datasets, including GTSRB and an aircraft taxing dataset, as well as comparing it to other proposed solutions, one section that really caught my eye was the comparison of the proposed sensitivity prioritized solution to a completely random one. The core concept of the paper was to propose that sorting by the sensitivity of an input feature (a pixel in all the examples given), the local optimal solution as described above, would provide a more explainable result. This makes a lot of sense, but what really caught my eye was just how random the random solution was. If you look at the figure below, my expectation was that some of the more prominent features would still be visible, and that would simply be a larger set for the random one. Well, there was a larger set, but there was incredibly little overlap of the two optimal sets, which really defied my expectations and begged the question, why?

## Why is random so random?

To start with, I forked the [VeriX repository](https://github.com/cram9030/VeriX) to begin exploring what might be happening there. I spent some time thinking about it and essentially what I theorized was happening is that as the irrelevant set grows the aggreigation of the pertubations increases approaching that $\delta$ threshold until it any additional pertubation would crest that threshold and everything afterwards would then be deemed *optimal*. The random sequence would have an opportunity to add more high-gain features earlier, which means it would reach this threshold faster than the sensitivity heuristic set. This was already somewhat supported by Figure 6 from the paper becuase it aligns with why the random *optimal* set was so much larger on average than the sorted one.

In an attempt to test that hypothesis I took a run of the MINST data set with the default parameters $\epsilon = 0.05$ and $\delta = -1e-6$ and plotted the histogram above which shows the number of *optimal* features/pixels given the size fo the irrelevant pixel set. What we see is that in both cases, there is a slight buildup, but then a point is reached where no more irrelevant pixels are found. In the random case, we see what we would expect, which is that there is a little more jittering, but the sensitivity heuristic has a very clean asymptotic approach. 

That still left me wondering, though, what that progression looked like. My intuition is obviously off, but I still expected there to be more overlap. If a cluster of pixels were truly important, would we not expect to see them involved, regardless of which local minimum we fall into? (Note: while this was my thought process, the real answer is probably not; this is a high-dimensional problem, and there are a lot of ways to turn a 2 into not a 2.) ## Conclusion
As I mentioned in the introduction, I began reading this paper with the intention of considering how I could apply the results. There are obvious limitations related to the type of model that can be used, but I actually don't think that is insurmountable; it just requires extra effort. What was really of interest to me was if I would be able to set those epsilons and deltas based off of underlying set variability in the case of $\epsilon$ or the allowable change to the output in the case of the $\delta$ which would enable a systematic way of analizing the model or in the best case being able to create requirements for the model that went a little deeper than an confusion matrix. After reviewing the paper, I found that there is nothing fundamentally incorrect, but it also does not verify something that I can act on. If anything, looking just at the sensitivity plots seems to extract all the information provided; however, at that point, I would probably reach for a saliency map.
